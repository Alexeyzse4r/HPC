$\Huge{Генетический алгоритм}$

Генетический алгоритм реализован в соответствии с документом(ТЗ/руководство). Реализация была выполнена на CPU и GPU для того, чтобы иметь возможность сравнить время 
и точность работы. В версии на CUDA используется curand для генерации случайных чисел на GPU. В реализации на GPU так же используется Thrust для параллельной сортировки популяции.
Обе реализации имеют следующий цикл работы: инициализация → оценка fitness → селекция → скрещивание → мутация.


В реализации используются следующие ядра:
setup_rng	- выполняет инициализацию генераторов случайных чисел для каждого потока
init_population -	выполняет инициализацию популяции
fitness_kernel -	выполняет оценку fitness (MSE/MAX_ERROR для всех особей параллельно)
crossover -	выполняет скрещивание
mutation -	выполняет мутацию

Исходная функция:
f(x) = 1.0 + 2.0 * x + 0.5 * x ^ 2 + 0.1 * x ^ 3 + 1.0 * x ^ 4;

По результатам видно, что с увеличением размера популяции увеличивается и ускорение GPU по сравнению с CPU - это весьма закономерное поведение. При недостаточно больших
размерах время на инициализацию и пересылки нивелирует ускорение GPU, занимая значительную долю общего времени выполнения.

По результатам так же можно сделать вывод, что коэффициенты заданного полинома восстанавливаются корректно, однако можно заметить, что наблюдается интересная особенность: при использовании
ошибки MSE погрешность существенно меньше, чем при использовании максимальной ошибки.
